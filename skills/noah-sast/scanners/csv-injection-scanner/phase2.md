### Phase 2: 동적 테스트 (검증)

Phase 1 소스코드 분석이 완료되면 후보 목록을 사용자에게 보여준다. 후보가 존재하는 경우에만 동적 테스트를 진행한다. 후보가 0건이면 동적 테스트 없이 바로 보고서를 작성한다.

**동적 테스트 정보 획득:**

사용자에게는 테스트 대상 도메인(Host)과 직접 획득이 불가능한 정보(외부 콜백 서비스 URL, 프로덕션 전용 자격 증명 등)만 요청한다. 그 외의 정보는 소스코드 분석 또는 관련 엔드포인트에 직접 HTTP 요청을 보내 획득한다.

사용자가 동적 테스트를 원하지 않거나 정보를 제공하지 않으면, 소스코드 분석 결과의 모든 후보를 "후보"로 유지한 채 Phase 3으로 진행한다.

**테스트 방법:**
1. 수식 시작 문자가 포함된 데이터를 입력
   - **테스트 대상 기능(댓글, 게시글, 연락처 등)에 기존 데이터가 없으면 이 단계에서 직접 생성해야 한다. 데이터가 없다는 이유로 Step 2로 바로 이동하거나 [환경 제한]으로 분류하지 않는다.**
2. CSV/Excel 내보내기 기능 실행
3. 다운로드된 파일에서 수식 문자가 이스케이프 없이 포함되어 있는지 확인

**테스트 페이로드:**
- `=1+1` — 기본 수식 (Excel에서 `2`로 계산되면 취약)
- `=HYPERLINK("https://CALLBACK_URL/csv-test","click")` — 외부 요청 유발
- `+cmd|'/C calc'!A1` — DDE 실행 시도
- `-1+1` — 음수로 시작하는 수식
- `@SUM(1+1)` — @ 접두사 수식

```
# 1. 수식 페이로드가 포함된 데이터 입력
curl -X POST "https://target.com/api/contacts" \
  -H "Cookie: session=..." \
  -H "Content-Type: application/json" \
  -d '{"name":"=1+1","email":"test@test.com"}'

# 2. CSV 내보내기 실행
curl -o export.csv "https://target.com/api/contacts/export?format=csv" \
  -H "Cookie: session=..."

# 3. 다운로드된 CSV 내용 확인
cat export.csv
# name,email
# =1+1,test@test.com  ← 이스케이프 없이 수식 포함 → 취약
# '=1+1,test@test.com  ← 작은따옴표로 이스케이프됨 → 안전
```

**검증 기준:**
- **확인됨**: 다운로드된 CSV/Excel 파일에서 수식 시작 문자가 이스케이프 없이 셀에 포함된 것을 직접 확인함
- **후보**: 동적 테스트를 수행하지 않았거나, CSV 내보내기 기능에 접근할 수 없어 수동 확인이 필요한 경우
- **"확인됨"은 동적 테스트를 통해서만 부여할 수 있다**: 소스코드 분석만으로는 아무리 취약해 보여도 "확인됨"으로 보고하지 않는다. 동적 테스트로 실제 트리거를 확인한 경우에만 "확인됨"이다. 동적 테스트를 수행하지 않았으면 모든 결과는 "후보"이다.
- **보고서 제외**: 수식 시작 문자가 이스케이프되어 있거나(`'` 접두사), CSV 내보내기 기능이 없는 경우

**개별 문자 검증 원칙 (중요):**
동적 테스트 시 수식 시작 문자(`=`, `+`, `-`, `@`)를 **각각 개별적으로** 테스트하고, 다운로드된 CSV 응답에서 **각 문자별로** 이스케이프 적용 여부를 확인해야 한다. 일부 구현에서는 특정 문자만 이스케이프하고 나머지는 누락하는 경우가 있으므로, "수식 문자가 이스케이프되지 않았다"고 일괄 보고하지 않는다.

- 각 페이로드의 CSV 출력을 개별 확인하여, 선행 `'`(작은따옴표)나 `\t`(탭) 등 이스케이프 접두사가 붙어 있는지 판별한다.
- 이스케이프가 적용된 문자는 "안전"으로, 이스케이프가 누락된 문자만 "취약"으로 정확히 구분하여 보고한다.
- 보고서에는 **실제로 이스케이프되지 않은 문자만** 취약 대상으로 기재한다.

예시:
```
# CSV 응답에서 확인된 결과:
=1+1       → '=1+1      (= 이스케이프됨 ✓)
+cmd|'...' → '+'cmd...' (+ 이스케이프됨 ✓)
-2+3       → -2+3       (- 이스케이프 없음 ✗ → 취약)
@SUM(1+1)  → '@SUM(...) (@ 이스케이프됨 ✓)

→ 보고: "-로 시작하는 수식만 이스케이프되지 않음"
→ 오보: "=, +, -, @ 수식 문자가 이스케이프되지 않음" ← 이렇게 쓰면 안 됨
```
