### Phase 2: 동적 테스트 (검증)


**테스트 전 준비:**
- 외부 콜백 수신 서비스가 필요한 경우 사용자에게 확인 (webhook.site, Burp Collaborator, requestbin 등)
- 테스트 환경에서 내부 네트워크 접근이 차단되어 있을 수 있으므로, 외부 콜백 방식을 우선 시도

**Basic SSRF 테스트:**
1. curl로 SSRF 페이로드가 포함된 요청을 전송
2. 응답에서 내부 서비스/메타데이터 정보가 반환되는지 확인
3. 일반적인 테스트 대상:
   - 클라우드 메타데이터: `http://169.254.169.254/latest/meta-data/`
   - 내부 서비스: `http://localhost:PORT/`, `http://127.0.0.1:PORT/`
   - 내부 네트워크: `http://10.0.0.1/`, `http://192.168.0.1/`

**Blind SSRF 테스트:**
1. 외부 콜백 URL(사용자 제공)을 페이로드로 사용
2. 콜백 서비스에서 서버로부터의 요청 수신 여부 확인
3. 콜백 서비스가 없으면 응답 시간 차이로 추정 (존재하는 내부 호스트 vs 존재하지 않는 호스트)

**URL 필터 우회 테스트 (필터가 존재하는 경우):**
- IP 표현 변형: `0x7f000001`, `2130706433`, `0177.0.0.1`, `127.1`
- URL 파싱 차이 이용: `http://evil.com@127.0.0.1`, `http://127.0.0.1#@allowed.com`
- DNS rebinding: `http://[공격자가 127.0.0.1로 resolve하는 도메인]`
- 프로토콜 변형: `gopher://`, `file://`, `dict://`
- Redirect chain: 허용된 도메인에서 내부 IP로 리다이렉트
- IPv6: `http://[::1]/`, `http://[0:0:0:0:0:ffff:127.0.0.1]/`

**검증 기준:**
- **확인됨**: 동적 테스트로 서버가 공격자 지정 주소로 요청을 보낸 것을 직접 확인함
