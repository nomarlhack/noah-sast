### Phase 2: 동적 테스트 (검증)

사용자에게는 테스트 대상 도메인(Host)과 직접 획득이 불가능한 정보(테스트 계정 2개의 인증 정보 등)만 요청한다. 그 외의 정보는 소스코드 분석 또는 관련 엔드포인트에 직접 HTTP 요청을 보내 획득한다.

IDOR 테스트에는 최소 2개의 서로 다른 권한/소유의 계정이 필요하다:
- 계정 A: 리소스의 정당한 소유자
- 계정 B: 리소스에 접근 권한이 없는 사용자

**수평적 IDOR 테스트:**
1. 계정 A로 인증하여 계정 A 소유의 리소스 식별자를 확인
2. 계정 B의 인증 정보로 계정 A의 리소스 식별자를 사용하여 요청
3. 계정 A의 리소스가 반환되면 IDOR 확인

**수직적 IDOR 테스트:**
1. 관리자/높은 권한 계정으로만 접근 가능한 엔드포인트와 리소스 식별자를 확인
2. 일반 사용자 계정의 인증 정보로 동일 요청 전송
3. 접근이 허용되면 수직적 권한 상승 확인

**리소스 열거 테스트:**
1. 순차적 ID를 사용하는 엔드포인트에서 ID를 순차적으로 변경하여 요청
2. 인증 없이 또는 다른 사용자의 인증으로 타인의 리소스가 반환되는지 확인

**검증 기준:**
- **확인됨**: 동적 테스트로 다른 사용자의 리소스에 실제 접근한 것을 직접 확인함
