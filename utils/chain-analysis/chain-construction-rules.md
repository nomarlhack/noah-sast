# 체인 구성 규칙

연계 분석에서 체인을 작성하거나 검증할 때 적용하는 5개 규칙.
한 체인이 아래 중 하나라도 해당하면 그 체인은 무효(폐기/약화)다.

## R1. 포함 관계
한 후보가 트리거되는 시점에 다른 후보의 능력이 이미 공격자에게
부여되면 무효. 임의 코드 실행은 navigation·DOM·세션 read를 모두 포함한다.

판정: "후보 A 트리거 시점에 후보 B의 능력이 이미 부여되는가?" YES → R1.

## R2. 정찰 결함
단순 코드 가시화는 다른 후보의 트리거 가능 여부를 바꾸지 않는다.
정찰은 공격 비용 절감 요인일 뿐 체인 단계가 아니다.

단, 노출 내용이 자격증명·토큰·내부 endpoint 등 실행 가능 데이터이면
R2 아님 — R4(데이터 흐름) 판정으로 넘긴다.

판정: 노출이 코드 가시화 → R2. 실행 가능 데이터 → R4 판정.
R2 결함은 "공격 표면 노트"로 독립 후보에 분류한다.

## R3. 동일 권한 잉여
두 후보가 같은 권한을 요구하고, 그 권한 자체로 이미 더 큰 직접 피해가
가능하다.

판정 (모두 YES → R3):
1. 두 후보의 트리거 권한이 정확히 같은가?
2. 그 권한 보유자가 체인 없이도 같거나 더 큰 피해를 낼 수 있는가?

## R4. 데이터 흐름 부재
한 후보의 출력값이 다른 후보의 입력 sink로 흘러가는 코드 경로를
파일경로:라인번호 2개 이상으로 제시할 수 없다.

판정:
1. Step N의 어떤 코드 출력이 Step N+1의 어떤 코드 입력 sink로 도달하는가?
2. 그 흐름을 파일경로:라인번호 2개 지점 이상으로 명시할 수 있는가?
둘 중 하나라도 NO → R4. "공격자가 X를 한 후 Y를 한다"는 행동 가정은
코드 흐름이 아니다.

## R5. Narrative 의존
"만약/~라면/~한다면" 가정 분기가 2개 이상이다.

판정: 가정 분기 단어 카운트.
- 0~1개: 해당 없음 (1개는 트리거 조건으로 허용)
- 2개 이상: R5

---

## 적용 결과 분류

- **통과**: 5개 모두 해당 없음 → 체인 유지
- **약화**: 일부 단계만 유효 → 유효 단계만 부분 체인/독립 후보로 분리
- **폐기**: 1개 이상 해당 → 체인 삭제, 관련 후보를 모두 독립 후보로
  분리. 사유 태그(R1~R5) 명시.
